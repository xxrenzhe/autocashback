# AutoCashBack V1 产品方案与实施方案

## 1. 品牌与定位

- 品牌名统一为 `AutoCashBack`
- 产品定位为返利网账号、Offer 与换链接运营管理平台
- 官网定位为对外获客型 SaaS，开户方式为管理员开通，不提供公开注册
- V1 只支持手工运营模式，不承诺返利网 API 接入

## 2. UI 规范

### 2.1 视觉方向

- 设计风格采用 `Editorial Cashback Ops`
- 主色使用 `emerald-600`
- 强调色使用 `amber-500`
- 背景色使用 `stone-50`
- 主要文字使用 `slate-900`
- 页面气质强调可信、稳定、专业，不使用夸张收益表达

### 2.2 字体与排版

- 营销页标题：`Noto Serif SC`
- 正文与后台：`Noto Sans SC`
- 金额、脚本 Token、统计数字使用等宽数字展示

### 2.3 页面结构

- 官网：首页 Hero、平台说明、工作流、核心模块、脚本说明、FAQ、底部 CTA
- 登录页：左侧价值说明，右侧登录表单
- 后台：左侧栏分 `用户区域` 与 `管理员区域`

## 6. 核心业务流程

### 6.1 用户运营流程

1. 管理员创建账号
2. 用户登录后台
3. 用户录入返利网账号
4. 用户手工创建 Offer
5. 系统定时解析推广链接终链
6. 用户复制 MCC 脚本到 Google Ads
7. 脚本按 `campaignLabel` 更新 Final URL Suffix

### 6.2 佣金阈值流程

- `commissionCapUsd` 默认值为 `200`
- 用户手工录入 `manualRecordedCommissionUsd`
- 当录入值大于等于阈值时，系统产生人工停投预警
- V1 不自动停投 Google Ads

## 7. 脚本方案

- 平台通过 `/api/script/link-swap/snapshot` 向 MCC 脚本提供只读快照
- 脚本通过 `X-Script-Token` 鉴权
- 脚本按 `campaignLabel` 找到目标 Campaign
- 只更新匹配标签的 Campaign 和 sitelink suffix
- 不使用 Google Ads API

## 8. 生产环境变量

生产环境最小化环境变量如下：

```bash
DATABASE_URL=
REDIS_URL=
JWT_SECRET=
ENCRYPTION_KEY=
NEXT_PUBLIC_APP_URL=https://www.autocashback.dev
INTERNAL_APP_URL=http://127.0.0.1:3000
NODE_ENV=production
TZ=Asia/Shanghai
DEFAULT_ADMIN_PASSWORD=
```

规则：

- 缺少 `DATABASE_URL` 或 `REDIS_URL` 直接启动失败
- 缺少 `JWT_SECRET` 或 `ENCRYPTION_KEY` 直接启动失败
- `DEFAULT_ADMIN_PASSWORD` 为明文初始密码，只用于首次管理员初始化

## 11. 验收标准

- 全站品牌名只出现 `AutoCashBack`
- 官网、登录页、后台均可访问
- 管理员可创建用户
- 用户可管理多个返利网账号
- 用户可创建 Offer 并绑定账号
- 调度器可解析终链并记录日志
- MCC 脚本可获取快照并按标签更新链接
- `manualRecordedCommissionUsd >= commissionCapUsd` 时生成预警
- CI 与镜像发布工作流可运行
