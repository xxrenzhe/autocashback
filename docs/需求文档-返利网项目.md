业务需求：

2.首页宣传页面：根据下面具体的业务需求，构建宣传首页
3.登录后，用户会进入后台管理系统，左边栏包括“用户区域”和“管理员区域”
4.“用户区域”需要包括如下功能：
1）仪表盘：参考autobb项目的仪表盘（页面/dashboard）设计，不需要“API配额使用”和“AI Token成本”等统计数据
2）账号管理：管理众多的返利网账号，需要支持不同的返利网平台，也要支持同一个返利网平台的多个不同账号，需要用户录入如下参数：
- 账号名：自定义返利网账号的名称，便于区分
- 注册邮箱：注册返利网平台的邮箱地址
- 提现方式：PayPal/银行转账（ACH）/礼品卡兑换
3）Offer管理：用户手动创建offer，需要填写的参数包括：
- 推广链接：返利网专属的推广链接
- 推广国家：推广的国家
- 品牌名：推广的商品的品牌名
- 返利网平台：归属哪个返利网平台
- 返利网账号：归属哪个返利网账号
4）换链接管理：定期访问推广链接，解析出最终链接，然后通过脚本执行实现Google Ads中链接的更换
- 依赖代理URL访问推广链接，获取最终链接
- 依赖MCC中配置的脚本的定时执行
- 初版脚本参考docs/changlink.md，根据产品需求进行完善和细化
- 不需要autobb中多种换链接方式，只需要引导用户复制换链接脚本即可
5）系统设置：
- 代理配置：直接复用autobb项目中页面/settings下的“代理设置”的功能
- 返利网配置：配置返利网的API和参数，用于接入多家返利网平台，比如Topcashback、乐天等
- 换链接配置：默认配置换链接脚本，用户可以直接复制，然后配置在MCC中
5.“管理员区域”需要包含如下功能：
1）用户管理：直接复用借鉴autobb项目的用户管理系统，包括登录页面（/login）、后台用户管理页面（/admin/users）等
